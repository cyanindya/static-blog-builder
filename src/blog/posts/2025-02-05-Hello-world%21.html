Preface

We all know how new projects should always be started by a "hello world" post, so I decided to make one as well.

This file serves to give you an example on how you should write your post before compiling the file into browser-readable HTML file.

As you see by now, the contents of your blog post should be written in HTML. The HTML file should be saved with name format denoting the post date and post title, although this is not mandatory. If you notice the first comment of this document, you see the full title of this post; the make-blog.js will use that comment as the post title, and if no date is given in the file name, the program will use current date.

If you want to create multi-page blog post, simply add <!-- break --gt; between two paragraphs. You should see the output file under dist/blog/posts/ already have the post paged.

By default, I use the bluesky-comments library to use existing Bluesky thread from my account as comment section of the post. If you'd like to use other services such as Disqus, you'll need to make adjustments to the template-blog-main.hbs file.

I think that concludes my brief write-up on how to use the static blog generator.

Have fun!
